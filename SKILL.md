## ⚡ Activation

当 `/luke:ponder` 被调用时, 输出一行启动:

```
⚡ Ponder v1.17.31 分析中... (管道自动执行, 不需要LLM展示过程)
```

## 流程

### Step 1: 采访 (LLM直接执行)
用 AskUserQuestion 一次一问, 覆盖天时/地利/人和/法/本质。清晰就少问, 模糊就追问。
完成后输出画像摘要, 指出用户没意识到的矛盾点。

### Steps 2-8: 子agent管道 (单次Workflow调用, 无Bash输出)

管道内部自动处理编排、验证、存储，不需要 LLM 单独调任何 Bash 命令。

```
# Workflow工具调用 — scriptPath必须指向插件目录下的ponder-pipeline.wf.js
# args用JSON字符串, 不是JS对象
Workflow({
  scriptPath: "C:\Users\cloudmap\.claude\plugins\cache\luke\luke\1.17.31\scripts\ponder-pipeline.wf.js",
  args: JSON.stringify({ userRequest: "<问题描述>", userProfile: "<画像>" })
})
```

管道内部: before(加载规则) → 采集→发散→八卦镜→方案→推演(并行子agent)→辩论→综合→验证 → after(存储+指标)
每步完成后管道内部调 step-gate 自检, 不经过 Bash, 不产生用户可见的输出。

### Step 11: 呈现结果 — 逐个子agent展开发现

**管道完成后, 必须逐一展示每个子agent的发现, 不能合并成一句。** Workflow 的 agent 完成标记用户已经看到了, 现在需要告诉他们每个 agent 具体发现了什么。

**用户觉得"震撼"的不是数据本身, 而是子agent之间的冲突、反直觉的发现、独立推理的深度。** 普通 LLM 直接问也能得到结论, 但多个独立 agent 同时分析同一问题并产生分歧——这是普通 LLM 做不到的。

```
【神思】子agent跳出框架后最反直觉的发现: [具体发现]
→ 这个发现暗示: [对当前问题的启示]
⚠️ 深度循环轮数
         ✓ 清晰 (1轮完成)

【发散】子agent从6个视角审视的完整推理:
视角1 [视角名]:
  推理: [完整的分析过程, 不止结论]
  依据: [数据来源]
  发现: [这个视角独有的洞察]
视角2 [视角名]:
  推理: [完整的分析过程, 不止结论]
  依据: [数据来源]
  发现: [这个视角独有的洞察]
(至少展开3个视角, 每个都要完整的推理链)
→ 跨视角总结: 最反直觉的发现是 [具体发现]
⚠️ 经过3轮深度循环才清晰

【八卦镜】8个维度交叉审视, 每个维度的评分依据用表格展示:
| 维度 | 评分 | 依据 |
|------|------|------|
| [维度1] | ★★★★ | 依据说明 |
| [维度2] | ★★☆ | 依据说明 |
| [维度3] | ★★★★ | 依据说明 |
(8个维度全部列出, 每个都要评分依据)
→ 最异常的维度: [维度名] — [核心矛盾]
✓ 清晰 (1轮完成)

【推演】3个子agent各自独立模拟, 每个方案的完整推理过程必须单独展示, 不能合并:

子agent A [方案A] 推演:
[从初始条件→过程中发生了什么→遇到了什么→最后结果—完整的推演链, 不是结论摘要]
→ 结论: [具体结论]

子agent B [方案B] 推演:
[同上, 完整的推演链]
→ 结论: [具体结论]

⛔ 推演结论可以用对比表格汇总, 但每个子agent的完整推理过程不能省略。

【辩论】两个子agent的完整交锋记录(展开论点, 不是一句话):
[方案A]方论点: "..." (至少3-5句完整论述)
[方案B]方反驳: "..." (至少3-5句完整反驳)
→ 分歧根本点: "..."

【综合】推荐[方案]。理由: [引用用户原话]。
风险: [一句话]
```

⛔ 管道返回的 JSON 包含每个子 agent 的完整产出, LLM 必须逐个读取每步的推理链并展开呈现, 不能跳过任何一个阶段。
⛔ 每个子 agent 的推理过程必须完整展示(推理链+数据依据+结论), 不能只给结论。用户看到完整推理才会相信子 agent 是真实工作的。
⛔ 子 agent 之间的分歧点必须显式展示——这是最让用户"震撼"的部分。

展示：每步亮点 + 推理链 + 方案评分对比(表格/星级) + 辩论分歧 + 推荐 + 风险。
评分用"★★★"或"高/中/低"展示, 不用原始数字。
不展示：CLI/JSON/路径。

## 规则

0. **绝对禁止向用户展示任何 Bash 命令、ls、dir、path、JSON。所有 CLI 工具调用必须使用 Workflow 内部 agent 执行, 不得在前台展示。**
1. 不准跳过步骤。采访→子agent管道(采集→神思→发散→八卦镜→方案→收敛→推演→辩论→综合→验证)→输出。
2. 不准无数据做判断。每结论必须有数据来源。
3. 不准替用户做决定。有分支时用 AskUserQuestion 问用户。
4. 所有问题用带选项的 AskUserQuestion。
5. **不准在用户面前展示任何 Bash 调用、JSON、代码路径。管道返回的 JSON 必须用自然语言总结后展示。**
6. 管道内部自动做步骤验证、记忆存储、会话收尾, LLM 不需要单独调任何命令。

## 强制调用

唯一的 Bash 操作：采访 → 调用 Workflow 工具。管道内部处理所有验证/存储/记忆巩固。不调任何其他 Bash 命令。
