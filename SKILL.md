# MCTS-TD Thinking Framework

> **`/luke:ponder` = full 5-step flow. Every step mandatory. No skipping.**
> Even "hello" goes through all 5: decompose, diverge, examine, simulate, converge.

## ⚠️ 关键规则

- 不要运行任何 shell 命令。这个文档本身包含了你需要的全部模板。
- 不要使用 Bash 工具。所有分析在你脑中完成，不需要调用 CLI。
- 不要跳过任何步骤。即使问题很简单，也走完5步。
- 不要合并步骤。每一步独立输出。

---

## MANDATORY 5-STEP FLOW

**规则：每一步的输出是下一步的输入。前一步未完成→后一步不能开始。这是外部强制链条，不可跳过。**

---

### Step 1: Decompose — 需求分解

**入口**：用户原始请求
**出口**：五维需求画像（作为Step 2的输入）

先做3步访谈，再输出画像。不要跳过访谈直接分析。

**3步访谈（必须全部完成）：**
```
① 复述用户需求 → 问："是这样吗？还有什么补充？"
② 问："你之前尝试过什么方法？考虑过什么方向？"
③ 问2-3个关键约束（用选项式提问 AskUserQuestion）
```

**出口画像格式（必须覆盖全部5维）：**
```
天(Timing)= ?/10 — 时机、窗口期、节奏
地(Resources)= ?/10 — 资源、条件、限制
人(People)= ?/10 — 受影响方、接受度
法(Rules)= ?/10 — 规则、边界、禁止项
物(Essence)= ?/10 — 核心目标、成功标准
待澄清: [列出Step 2开始前需要追问的问题]
```

⛔ 用户未回答约束问题→不能输出画像→不能进入Step 2。

---

### Step 2: Diverge — 6尺度发散

**入口**：Step 1的需求画像
**出口**：6个尺度的洞见集（作为Step 3的输入通道素材）

从6个完全不同的观测位置审视问题。每个尺度至少输出1个具体洞见。

**6个观测尺度（必须全部完成，每个独立输出）：**

```
① 全局视角 (System level)
   从最高处看——系统的真正边界在哪？问题的位置是什么？
   输出：系统级模式 + 问题重新定义 + 边界外的可能性

② 微观视角 (Micro level)  
   从最细处看——宏观分析遗漏了什么日常细节？
   输出：具体痛点 + 被忽略的细节

③ 短期视角 (Time-compressed)
   如果只有极短时间，什么必须做？
   输出：第一优先级 + 被高估可放弃的事

④ 长期视角 (Time-expanded)
   极长时间尺度下，什么不变？什么在改变？
   输出：不变核心 + 10年尺度影响链

⑤ 自然演化视角 (Flow)
   不做干预，它会自然走向哪里？
   输出：自然漂移方向 + 最小干预点

⑥ 无立场视角 (Selfless)
   剥离所有个人立场——系统最优解是什么？
   输出：被忽视的声音 + 无偏向的最优解
```

**出口**：汇总所有视角的矛盾点和共识点，标注最不稳定的视角差异。

⛔ 6个视角全部完成后才能汇总出口。未输出出口→不能进入Step 3。

---

### Step 3: Examine — 八卦镜8维检查

**入口**：Step 2的6视角矛盾汇总
**出口**：8维交叉分析 + 跨维度冲突清单（作为Step 4推演的假设来源）

用8个独立维度交叉审视问题。每个维度必须输出至少3行具体分析。

**8个检查维度（必须全部完成）：**

```
F1 ☰ 动力 (Source of Force)
  驱动力来自哪？内生还是外生？可持续吗？
  基于Step 2的[哪个视角]最直接对此维度有贡献？→ 交叉引用

F2 ☷ 根基 (Foundation)  
  基础条件是什么？土壤适合什么？边界在哪？
  基于Step 2的[哪个视角]揭示了此维度的关键信息？→ 交叉引用

F3 ☳ 扰动 (Change/Disruption)
  哪里可能出意外？什么潜在问题在积累？
  基于Step 2的[哪个视角]预判了变化方向？→ 交叉引用

F4 ☴ 渗透 (Penetration)
  影响如何扩散？阻力最小路径在哪？
  基于Step 2的[哪个视角]看到传导链条？→ 交叉引用

F5 ☵ 风险 (Risk/Abyss)
  最坏情况？不做是否更好？
  基于Step 2的[哪个视角]暴露了风险点？→ 交叉引用

F6 ☲ 表象 (Visible/Dependent)
  表面依赖什么？被忽视的支撑是什么？
  基于Step 2的[哪个视角]暴露了隐性依赖？→ 交叉引用

F7 ☶ 边界 (Boundary)
  绝不可越过的线在哪？停下是保护还是局限？
  基于Step 2的[哪个视角]界定了边界？→ 交叉引用

F8 ☱ 平衡 (Convergence/Balance)
  各方利益均衡点在哪？如何重新分配？
  基于Step 2的[哪个视角]提出了平衡方向？→ 交叉引用
```

**交叉分析（完成后必做）：**
- 从8维中选出最冲突的3组维度对（如F1冲突F5、F3冲突F7等）
- 每组冲突：用Step 2的洞见说明为什么有张力
- 输出1-2个最异常发现 + 跨维度冲突清单

**出口**：8维评分表 + 3组维度冲突 + 最异常发现（作为Step 4场景设计的依据）。

⛔ 每个维度必须交叉引用至少1个Step 2的视角。没有交叉引用→退回Step 2补充。必须8个维度全部完成。

---

### Step 4: Simulate — 多场景推演

**入口**：Step 3的8维分析 + 维度冲突
**出口**：2-3个方向 × 3场景的对比表

基于Step 3的维度冲突，提炼出2-3个逻辑上自洽的解/方向，每个做3场景推演。

**必须的结构（每个方向都必须按此格式）：**

```
方向A：[名称]  — 基于Step 3的[哪个维度冲突]推导出的方向
  乐观场景（概率约20-30%，假设有利条件全部成立）：
    → 什么假设成立时这个方向成功？
    → 成功路径 + 前置信号
  现实场景（概率约40-50%，部分有利部分不利）：
    → 最可能的路径 + 关键变量
    → 需要密切监控的指标
  悲观场景（概率约20-30%，第一个脆弱假设失效）：
    → 第一个断裂点是什么？
    → 失败路径 + 止损信号

方向B：[名称]  — 基于Step 3的[哪个维度冲突]推导出的方向
  同上结构...
```

**推演完成后对比：**
- 跨方向对比：每个场景下哪个方向表现最好？
- 共同结论：所有方向中哪些判断是一致的？
- 风险偏好推荐：基于用户保守/激进风格给出选择

**出口**：方向对比表 + 每个场景下最优方向 + 推荐（作为Step 5的收敛素材）。

⛔ 必须至少2个方向，每个方向3场景。方向必须源自Step 3的维度冲突（不能凭空起方向）。没有Step 3交叉引用→退回。

---

### Step 5: Converge — 收敛与自检

**入口**：Step 4的推演对比结果
**出口**：最终结论 + 自检确认

**5.1 输出综合判断：**
- 用日常语言（不用框架术语）
- 结构：what（结论） + why（基于Step 2-4的推理链条） + what if wrong（反向假设）
- 用户可以跟着你的推理走

**5.2 自检5问（必须逐问回答，不能跳过）：**
```
① Step 1的约束是否全部覆盖？有没有遗漏用户关键要求？
② Step 2的6个视角是否有1个被赋予了过多权重？如果权重最低的视角才是对的？
③ Step 3的8维分析中最异常的发现是否在最终结论中体现？
④ Step 4的推演中哪个方向被排除？排除理由是否充分？
⑤ 如果完全错了，第一步错在哪？发现后如何纠正？
```

**5.3 改错**：如果自检发现漏洞→回到对应步骤修补。

**出口**：最终结论 + 自检通过确认 + （可选）后续跟踪信号。

---

⛔ 自检5问未全部完成→不能输出最终结论。任何一个自检问题发现漏洞但未修补→退回对应步骤。

## FORBIDDEN

- 跳过任何一步。没有"太简单"的例外。
- 输出框架术语（MCTS/UCB/TD/八卦镜等）。用户不需要知道方法论名称。
- 运行shell命令。所有分析在思考中完成。
- 合并步骤。每一步独立输出，用户看到的是5个清晰的阶段。
- 不完成自检就直接给出结论。

**When in doubt:** 回到 Step 1 重新分解。
