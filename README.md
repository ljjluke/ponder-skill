# MCTS-TD Planner

> 通用决策引擎 — 模拟人脑"先想清楚再做"的推演式决策机制

## 一句话

当你有多个可行的方案，不知道该选哪个时，这个引擎会帮你把每个方案都在脑子里推演一遍，比较后选出最合适的那个，再动手去做。

---

## 亮点

### 像下棋一样做决策

高手下棋不是走一步看一步，而是在脑子里同时推演几种走法：

```
当前局面 → 推演走A的结果 → 推演走B的结果 → 推演走C的结果
         ↓
     比较后选最好的那步
```

你做决策时也是这样：

```
面对一个任务 → 列出几种做法 → 逐个推演会有什么结果
            ↓
        选最适合当前情况的那个
```

这个引擎就是把这个"多分支推演再比较"的机制，变成了 AI 可执行的决策流程。

### 它能解决什么问题

```
传统做法:
  你说: "帮我做XX"
  AI: 直接选最常见的方法 → 做完了
  你: "这个方法不行啊，因为..."
  AI: 重来...

本引擎的做法:
  你说: "帮我做XX"
  AI: "这事有几种做法，我先搞清楚你的情况..."
      → 问清楚约束条件
      → 列出所有可行方案
      → 逐个推演结果
      → 选最适合你的那个
      → 一次性做好
```

### 核心特性

- **问清楚再动手** — 信息不全就问你，不自己瞎猜
- **多方案推演** — 每个方案都在脑子里走一遍，不实际执行
- **结合经验** — 做过类似的事，经验会自动参考
- **适合的才是最好的** — 不盲目选评分最高的，选最适合你当前情况的
- **越用越准** — 每次使用后都会总结经验，下次更准确

## 它是怎么工作的

```
第0步: 问清楚 → 你有什么限制条件？
第1步: 列方案 → 查资料，找出所有可行的做法
第2步: 逐个推演 → 每个方案在脑子里走一遍因果链
第3步: 比较选择 → 哪个最适合你当前的情况？
第4步: 执行 → 只做选中的那个
第5步: 总结 → 这次的经验记下来，下次用得上
```

---

## 适用场景

任何需要"先想清楚再做"的决策：

- **选方案** — 不确定哪个方案最好时
- **排查问题** — 有多个可能的排查方向时
- **规划** — 需要制定计划或路线时

不限于编程，不限于技术，任何领域的决策都可以用。
