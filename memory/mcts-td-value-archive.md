## 使用说明

### 状态说明

| 状态 | 权重 | 含义 |
|------|------|------|
| CONFIRMED | 1.0 | 可信知识，≥3次验证通过 |
| PROVISIONAL | 0.3 | 待验证，仅1-2次验证 |
| DISPUTED | 0.2 | 有矛盾证据，保留但低权重 |
| REFUTED | 0.0 | 已证伪，不参与召回 |
| HYPOTHESIS | 0.1 | 新知识，低权重参与召回（仅用于探索） |
| SLEEPING | 0.3× | 超过30天未使用，权重减半 |
| ARCHIVED | — | 超过90天未使用，不参与常规召回 |

### 召回方式

多路召回流程：
1. **精确匹配**：6维特征完全一致 → 权重1.0
2. **主维度匹配**：task_type + domain → 权重0.8
3. **领域匹配**：domain + risk_level → 权重0.5
4. **标签匹配**：tags 关键词重叠 → 权重0.3
5. **关联扩散**：已召回条目的关联关系 → 权重0.1~0.04

综合评分 = 路径权重 × 状态权重 × 时效衰减 × 上下文匹配 × 使用频率
取 top-5 注入推演。

### 更新方式

执行完成后，根据实际结果更新知识图谱：
1. 执行多路召回，找到匹配的知识条目
2. 如果找到 → 更新匹配条目的 n, q, σ², 状态, 巩固分
3. 如果没找到 → 创建新 HYPOTHESIS 条目（含 tags）
4. 记录变更到知识变更日志
