## ✨ 为什么不一样

大多数 LLM 工具直接回答——大概率答偏。Ponder 在回答前激活**完整思考回路**，每一步**代码强制**执行，不是靠提示词约束。

```
┌─ 用户提问 ──────────────────────────────────────────┐
│                                                       │
│  需求拆解 → 发散 → 八卦镜 → 方案                    │
│           → 推演 → 辩论 → 综合 → 验证                │
│                                                       │
│  每步：清晰？→ 继续。不清晰？→ 深度循环。            │
│  最终：有数据支撑的结论 → 记录为经验。                │
│                                                       │
└──────────────────────────────────────────────────────┘
```

### 🎯 核心亮点

| | 特性 | 为什么重要 |
|---|---|---|
| 🎯 **数据驱动** | 每个结论必须有数据来源。LLM 不准编。 |
| 🔄 **代码深度循环** | 不清晰就继续挖，没有上限。LLM 无法跳过。 |
| 🚫 **步骤强制** | 7个阶段全部必须执行，代码检查不是规则提醒。 |
| 💡 **错误收敛** | 记录失败→避免重复→越用越聪明。 |
| 🌍 **领域无关** | 金融、医疗、战略、技术——同一个框架。 |
| 🧘 **中国古文化根基** | 庄子视角、八卦镜、五行画像。 |

## 🔄 思考回路

### 第一步：需求拆解

```
AskUserQuestion 螺旋采访 → 覆盖5维度：
  天时 · 地利 · 人和 · 法 · 本质

输出：需求画像 + 待验证假设
```

### 第二步：分析（一次 Workflow 调用）

所有7个阶段在 `ponder-pipeline.wf.js` 内自动运行。LLM 调一次等结果。

| 阶段 | 做什么 | 关键字段 |
|------|--------|----------|
| **发散** | 6个不同视角审视问题 | `data_source`, `assumption` |
| **八卦镜** | 8维度系统评分 | `evidence`, `uncertainty` |
| **方案** | 5-8个具体可执行方案 | `condition`, `condition_verified` |
| **推演** | 每个方案独立模拟（并行） | `optimistic`, `neutral`, `pessimistic` |
| **辩论** | 方案排名+交叉论证 | `pros`, `cons`, `synthesis` |
| **综合** | 最终结论+推理链 | `conclusion`, `reasoning`, `pending_user_questions` |
| **验证** | 独立审查找漏洞 | `verdict`, `issues`, `fake_clarity` |

每步输出：
- `is_clear` — 结果够清晰了吗？
- `user_questions` — 这一步发现了什么盲点？

### 深度循环（代码强制）

Workflow 解析器不支持 `while`/`for`，用展开的 `if` 实现：

```
第1轮 → is_clear? → 是 → 下一步
       → 否 → 携带盲点到 →
第2轮 → is_clear? → 是 → 下一步
       → 否 → 携带盲点到 →
...
第5轮 → 最后一轮 → 不管清不清都继续
```

LLM 无法跳过、无法减少轮数、无法忽略 `user_questions`。

### 第三步：呈现前先确认

管道返回后，LLM **必须先确认**：
- `pending_user_questions` 有内容？→ 问用户
- 分析中提到的红线/条件触发了？→ 问用户
- 结论依赖的假设用户同意？→ 问用户

确认完才呈现最终结论。

## 💡 错误收敛（不是记忆系统）

不是通用记忆。**只记录失败，避免重蹈覆辙。**

```
做决策 → 出结果 → 存为经验
                   ↓
下次类似情况 → 查历史失败 → 排除已知死路
                               ↓
                        减少试错，越来越稳
```

### 经验格式

```
场景：当时什么情况
尝试：做了什么
关键条件：当时的环境因素
失败原因：为什么不行
替代方案：应该怎么做
```

### 存什么

- ✅ 失败案例、踩坑记录、用户纠正
- ❌ 市场价格、统计数据、新闻（会过期）
- ❌ 用户偏好（直接问就行了）
- ❌ 模型自带知识（LLM 本来就知道）

后台守护进程 `memory-monitor.js` 全程运行，自动捕获经验、去重合并（新的更好的信息会替换旧的）。
