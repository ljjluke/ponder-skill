## 信号过滤 — 从经验之流中识别值得注意的信号 (Signal Filter)

**学术来源**: 詹姆士(William James)「经验之流」(stream of experience) + 怀特海(Whitehead)「摄入」(prehension)。

### 为什么要这一步

传感器在持续推送信号（股价变动、成交量异动、新闻、社交媒体热度等）。如果每个信号都触发完整的9步推理管线，框架会被噪音淹没——大多数信号没有信息增量，不值得启动深度思考。

信号过滤层的职责是：**在信号流里识别"值得注意的东西"，只放行有信息增量的信号进入管线。**

这个层解决一个结构性缺陷：当前框架只有"思考段"（收到信号后怎么想），没有"感知段"（什么信号值得想）。没有过滤层，传感器是噪音源。

### 哲学底座

- **詹姆士「经验之流」**: 意识不是离散的认知状态，是连续流动的经验。传感器信号就是框架的经验之流——问题不在"怎么处理每个信号"，在"在这条流里怎么识别一个值得注意的东西"。
- **怀特海「摄入」**: 每个实际实有在"摄入"环境中其他实有的信息。摄入是有选择的——不是所有信号都进来，只有"对这个主体有意义的"才被摄入。什么有意义？由主体已有的"关心事项"决定。

### 门控

```
① 有传感器信号流在推送 → 启用信号过滤
② 无传感器信号流（用户直接提问） → 跳过过滤层，直接进入需求打磨（五诊）
```

### 操作：信号过滤三步

```
① 信号分类：把传感器信号分类为——
   - 异常信号（偏离基线的数值/事件，如单日成交量超过20日均量的3倍）
   - 机会信号（暗示行动窗口的信息，如关键均线金叉/死叉、财报超预期）
   - 矛盾信号（与当前画像/上次立场冲突的信息，如北向资金流向与价格走势背离）
   - 噪音（无信息增量，忽略）

② 摄入判断：对非噪音信号做摄入评估——
   - 这个信号是否与某个已有的"关心事项"相关？（画像里的约束/偏好/风险维度）
   - 这个信号是否暗示某个"上次判断"的前提被改变了？（读取 framework_self 的 stance_memory）
   - 这个信号是否在某个"已知盲点"的方向上？（读取 framework_self 的 pattern_memory）
   三个问题全否 → 低优先级，标记但暂不触发

③ 触发决策：满足任一条件 → 启动9步管线
   - 异常信号 + 高赌注（不可逆/高成本/涉及他人/长期影响）→ 必启动
   - 矛盾信号 + 与上次立场直接冲突 → 必启动（立场被现实数据挑战）
   - 机会信号 + 时间敏感（窗口可能关闭，如财报前/重大政策发布前）→ 必启动
   - 异常 + 低赌注 → 标记，累计三次同类型再启动（防过敏）
   - 噪音 → 不启动，不标记
```

### 与已有模块的边界

- **信号过滤 vs 神思前提审视**: 信号过滤管"这个信号值不值得想"，神思管"这个问题本身有没有伪前提"。过滤在前，神思在后——过滤放行后，神思才开始审视。
- **信号过滤 vs 八卦镜盲点**: 信号过滤读 framework_self 的 pattern_blindspot 判断"这个信号是否在已知盲点方向上"；八卦镜是发现新盲点的过程。
- **信号过滤 vs 需求打磨五诊**: 五诊是"用户画像构建"，信号过滤是"外部信号筛选"。过滤层放行后进入五诊（如果用户提问）或直接进入神思（如果信号触发了管线）。

### 对象层级

信号过滤不做"这个信号意味着什么"（那是9步管线的活），只做"这个信号值不值得启动管线"。
