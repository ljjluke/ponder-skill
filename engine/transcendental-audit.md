## 先验自检 — 评分工具的先验框架质询 (Transcendental Self-Audit)

**学术来源**: 康德《纯粹理性批判》先验演绎(A95-A130/B129-B169) + 范畴只能作经验性使用(A238/B298) + 先验幻相划界(A293/B349起)

**为什么要这一步**: 本管线的评分靠一套"先验框架"——8个维度(哪些方面算"好")+ 各维度权重(各占多少)+ 隐含的"好"的定义(可行性取什么时间尺度/稳健对抗什么扰动)。这套框架在所有具体评分之前就被给定,规定了"什么算好方案",而任何具体方案的分数都已被它先行规定了可被言说的维度与权重。问题是:这套框架本身从不进入推理被质疑——它从哪来?凭什么适用于当前这个case?当一个权重是从某类历史case归纳出来的(或纯粹文化隐喻拍板的),却拿来当"对当前case也必然成立"的真理用,这就是康德说的"先验幻相"——把经验性的东西越界当成了先验真理。先验自检是评分阶段的元层出口:在用尺子量之前,先质问尺子本身。

### 与已有五家族的运动方向差异(防换皮)

五大家族的运动对象:
- 自我否定(黑格尔):在框架内从矛盾推进求合题
- 归因可谬(皮尔士):在框架内从结果回溯原因
- 目的论(亚里士多德):在框架内从终态反推路径
- 他者性(列维纳斯):在框架内保留不可同化的异质项
- 语言放假(维特根斯坦):跳出"用户问题"的提法框架到元层消解问题

**先验自检质疑的是"评分工具"的先验框架**——和语言放假同属"质疑框架本身"但对象层级不同:
- 语言放假:框架在**输入端**(用户问题的提法),改的是问题陈述
- 先验自检:框架在**评分端**(管线的维度集/权重/聚合方式),改的是评分工具

用康德自己的区分:语言放假质问"这个问题被认识前预设了什么"(问题侧先验),先验自检质问"我们去评判时用的范畴凭什么合法"(认识侧先验)。两个不同的先验层。

### ⛔ 边界:与语言放假(维特根斯坦)的边界

这是最易混淆的点,必须写死:

两者可能 converge on the same word——比如用户问题里出现"可行",维特根斯坦消解"可行"这个空转词(改问题),先验自检质问"可行性维度凭什么进评分"(改工具)。**判据看动作落点**:
- 若质疑指向"用户的提法预设了不该预设的东西" → 语言放假,改问题陈述
- 若质疑指向"管线的评分维度/权重对该维度作了越界使用" → 先验自检,改评分工具
- ⛔若先验自检最后只是说"这个词在用户问题里有问题"而不动评分工具 → 退化成语言放假,应驳回回 dissolve-frame

**空转范畴 vs 空转词的手法区分**:先验自检的"空转范畴"判据是**在本case里找不到可观察的经验锚点来给它赋值**(A238/B298 经验性使用判据);语言放假的"空转词"判据是**在日常语言场景里找不到稳定用法**(PI §116 语用判据)。手法相似但判据来源不同(认识论 vs 语用),共存不混用。

### 门控 — 只在评分工具的合法性可疑时触发

```
① 触发条件(满足任一,且须已进入评分/收敛阶段):
   a. 幸存方案分数接近、排名对权重微小变动敏感(说明权重在决定结果而非方案差异)
   b. 分数高但用户/领域专家直觉不接受(说明尺子与对象不匹配)
   c. 当前case类型与权重的历史来源case类型不同(权重可能越界使用)
   d. 发现某维度在本case里无法被任何可观察经验锚点赋值(空转范畴)
② 不触发: 可逆小事、分数有明显梯度且与直觉一致、首次评分且case类型常见
③ 触发后: 先确认评分已执行(8维分已出),否则回评分阶段
```

这一步是评分阶段的元层兜底,不是每次评分都做。多数case的评分框架是适配的——只有评分结果与直觉背离、或case类型异于权重来源时才质疑尺子。

### 操作步骤 — 先验自检五步

```
① 枚举先验框架: 把当前评分工具的先验层显式列出——
   (a) 维度集(哪些维度被纳入"好",为什么是这几个不是别的)
   (b) 各维度权重(各占多少,为什么是这个数)
   (c) 聚合方式(线性加权/乘法门/其他,为什么这么聚合)
   (d) 隐含的"好"的定义(如可行性默认取什么时间尺度、稳健默认对抗什么扰动)
   ⛔这一步只是上桌,不评判。

② 逐项问"何以必要"(正向): 对每个维度/权重/聚合方式问——
   "若删掉这一项,本case的判断会变成什么样?"
   - 删掉后仍能得出可辩护判断 → 该项对本case是经验性附加而非先验必要,标可质疑
   - 删掉后判断坍塌(无法排序/无法区分) → 该项是本case的真正先验条件,保留

③ 逐项问"何以可能"(反向): 对每个保留的先验项问——
   "这个维度的可度量性来自哪个经验锚点?" 若在本case里找不到任何可观察的
   经验锚点给它赋值,标为"空转范畴"(范畴只能作经验性使用,越界即幻相)。

④ 划界测试(幻相检测): 问——"当前这套维度/权重,是否在越界作超验使用?"
   即是否把一个只在某类历史case验证过(或纯粹文化隐喻拍板)的权重,
   当作对当前这个新case也必然成立的真理? 对每个权重标注其经验来源case类型,
   若当前case类型不在其中 → 标"越界使用"。

⑤ 输出(三选一,不准敷衍):
   (a) 框架通过自检 → 维持原评分,记录"本次先验自检已执行"
   (b) 部分维度/权重在本case失效 → 输出"维度X在本case无经验锚点/权重Y越界,
       本case改用[Z]"并给Z的经验依据,改用结果须能被验证阶段检验
   (c) 整套框架对本case不适配 → 触发"暂停评分,回问题阶段重判",
       不准在原框架内硬排
```

### 与 weights.js 学习机制的关系(方向相反不冲突)

weights.js 的 learn 是从经验结果**反向调整权重数值**(经验性归纳);先验自检是**质问这套归纳出的权重是否被越界当成了先验真理**。learn 调数值,先验自检质问数值的合法性,两者方向相反不冲突——一个在调尺子刻度,一个在问"用这把尺子量当前对象"本身是否合法。

### 对象层级

先验自检只管"评分工具的先验框架是否合法",⛔不上升到"质疑用户问题提法"(那是语言放假的活),也不上升到"质疑结论对错"(那是可谬标注的活)。它处在评分节点,对的是管线自己的尺子。

**兜底**: 即便自检不准,"必须枚举先验层+必须为每项找经验锚点+越界项必须标注来源case"的约束保证了它不空转——至少把一个被默认的权重或维度送上桌,或至少承认"这把尺子没被验证过适用于本case"。

⛔ 不引入哲学术语(先验/范畴/演绎/幻相/康德)。对外输出用自然语言:说"这套评分标准默认了X维度最重要、且X在本case可能没法客观打分——要么给X找个能落到具体场景的衡量方式,要么本case换个评分口径",不说"对评分范畴作先验演绎以检测越界使用导致的先验幻相"。
