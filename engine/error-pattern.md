## 错误归类 — 从数值学习到认知学习 (Error Pattern Classification)

**学术来源**: 贝特森「学习层级」(Bateson, 1972) + 库伯「学习循环」(Kolb, 1984) + 弗拉维尔「元认知」(Flavell, 1979) + 道家「为道日损」(《道德经》48章)。

### 为什么需要这一步

当前进化引擎的学习是 **Learning 0**（贝特森层级最低阶：改反应阈值）——`verify 失败 → 某系数 +0.2`，学的是数值。它记不住"这次推理在哪类问题上犯了哪类思维错误"，下次同类问题仍会重蹈。

错误归类层的职责：**把行为信号（清晰度低/问题多/被驳斥）归类成可命名的推理偏差类型 + 对应检查项，让学习从"调数值"升级到"记认知模式"**（贝特森 Learning II：改"我如何选择行为"的规则）。

这是库伯学习循环里**缺失的"反思观察"段**——当前只有"经验(verify fail)→概念化(权重+0.2)"，缺"这次错在哪一步、错属哪类"的反思，调参就没有认知锚点。

### 哲学/学科底座

- **贝特森「学习层级」**: L0=改反应阈值（当前状态）；L1=在同情境修正行为；L2=改"我如何选择行为"的规则（情境归类的修正，错误归类做这个）；L3=改"我如何归类情境"这套元规则。当前框架只做 L0，错误归类补 L2。
- **库伯「学习循环」**: 经验→反思观察→抽象概念化→主动实验。当前缺"反思观察"段——失败后没产出"错属哪类"就跳到调参，归类补这一段。
- **弗拉维尔「元认知」**: 元认知=对认知的认知。学习信号应是"自评置信 vs 实测结果"的差，而非对错本身——"自评高却错"的 case 才是高价值学习样本（对应邓宁-克鲁格：无法发现自己不知道是能力缺陷同源）。
- **道家「为道日损」**: 学知识是加法，修道是减法。学习库不能只增不删——长期不命中的旧规则、被证伪推翻的旧经验要定期删除，否则加法学习退化为噪音。

### 归类映射：step × 信号 → 推理偏差类型 + 检查项

基于真实翻车数据（集中在 divergence/bagua 两步，主因"问题多+清晰度低"），把行为信号归类到可命名的推理偏差：

| 步骤 | 行为信号 | 归类为(推理偏差类型) | 检查项(下次同类问题触发) |
|---|---|---|---|
| divergence(发散) | 问题多+清晰度低 | **视角覆盖不全**(漏看关键视角，靠追问补) | 下次发散起跑前先列"必覆盖的6视角清单"，逐个打勾再产出 |
| divergence | 清晰度高但后续被驳 | **过早收敛**(六视没互否就汇共识) | 下次共识前强制找1-2对对立视角互质疑 |
| bagua(八卦镜) | 问题多+清晰度低 | **盲点扫描失焦**(维度 agent 各自为政没汇总) | 下次每维度 agent 产出后必须交叉引用其他维度 |
| bagua | key_finding 被驳 | **盲点误判**(把非盲点当盲点) | 下次 key_finding 产出前先问"这个真的是别人看不到的吗" |
| plans(方案) | 方案被驳/被淘汰多 | **可行性锚定**(只从约束内推方案没反向链终态) | 下次方案起前先做终态画像反向链 |
| converge(收敛) | 淘汰理由被驳 | **骑墙不淘汰**(没立场，留太多方案) | 下次收敛必须明确倾向+凭什么 |
| simulate(评分) | 分数被驳/排名翻转 | **评分先验错位**(权重来源与case类型不匹配) | 下次评分前先验自检权重是否适用本case类型 |
| synthesis(综合) | 结论被驳/可谬命中 | **结论自反缺失**(没质疑共享前提) | 下次综合强制做结论自反+可谬标注 |
| 任意步 | 自评清晰度高+实测被驳 | **元认知偏差**(邓宁-克鲁格：高估自己) | 下次该步产出前先列"我对自己这步结论有多自信+依据"，置信高但依据弱=危险 |

### 三种学习信号的价值分级（波普尔可证伪 + 弗拉维尔元认知）

不是所有失败都值得学。按波普尔"证伪比证实携带更多信息"+弗拉维尔"自评-实测差"分级：

```
高价值学习样本（必入库 + 触发归类）:
  - 自评清晰度高 + 实测被驳（元认知偏差，邓宁-克鲁格信号）
  - 同类型问题反复在同一类判断上翻车（贝特森L2信号，二阶模式）
  - 用户明确驳斥(REFUTED)（波普尔证伪，最高信息量）

中价值（记但不优先）:
  - 清晰度低 + 问题多（L0信号，只知"不准"不知"错哪类"）
  - verify 失败无 issues（泛化失败，信息量低）

低价值/不学（防污染）:
  - 成功例（波普尔：证实不携带认知价值，只作低权重背景）
  - 远超能力的越界失败（维果茨基ZPD外，是噪音不是教训）
  - 偶发异常（未重复出现的，可能是 case 特殊不是推理错）
```

### 离线巩固阶段（记忆科学：睡眠依赖巩固 + 再巩固）

当前学习是逐 run 即时调权（在线），没有离线巩固。记忆科学说长期固化靠离线批量回放：

```
离线巩固（跑完一批 run 后触发，不是逐 run）:
  ① 批量回放失败案例，提取跨案例共性（"3个case都在区分相关vs因果栽"）
  ② 共性固化为归类规则（写入本表）
  ③ 单 case 只作短期缓存，共性才固化为长期规则

再巩固(reconsolidation):
  - 回放旧案例时若当前 run 结果与旧案例冲突，就地更新该案例
  - 不新建一条（否则同型错误在库里分裂成多条矛盾记录）
  - 测试效应(testing effect): 新问题先逼推理链自问"上次类似问题怎么错的"
    再比对——提取动作本身才巩固，重读不巩固
```

### 道家日损机制（为道日损 + 反者道之动 + 坐忘）

```
日损（定期删除，防加法退化）:
  - auto-fix 规则长期未被回放命中 → 删除（不是保留成噪音）
  - 被新证伪例推翻的旧归类规则 → 标记失效或删除
  - 同类问题已有N条规则，新增前先看能否合并而非追加

反者道之动（饱和反转预警）:
  - 某系数被反复+0.2到阈值（如0.8+）→ 标记"可疑过拟合"
  - 停止继续强化，触发反向校验（是不是该减而不是加）

坐忘（清空当前策略的逃逸阀）:
  - 连续多 run 套用同一策略仍失败 → 丢弃当前策略栈从零重判
  - 而非在烂策略上继续调参（这是道家对"沉没成本"的解药）
```

### 与已有模块的边界

- **错误归类 vs evolve.js 权重学习**: 权重学习是 L0（调数值），错误归类是 L2（记认知模式）。两者并存：权重管"这个信号多可信"，归类管"这次错属哪类"。归类是权重的认知锚点。
- **错误归类 vs auto-fix**: auto-fix 当前记"清晰度低→加预备步骤"（L0 机械修复）。错误归类给它加 `error_pattern` 字段——从"加步骤"升级到"记哪类错+触发哪个检查项"。
- **错误归类 vs recallErrors**: recallErrors 记"用户驳斥(REFUTED)"是刺激-反应。错误归类加一层"这次被驳是因为哪类推理偏差"。
- **错误归类 vs outcome-learning.md(未落地)**: outcome-learning 是 feature 分支四层大脑的"结果学习层"，设计了贝特森二阶但未合入 main（见 [[brain-loop-architecture]]）。错误归类是 main 上对"二阶学习"的最小可验证落地——不重写，补既有 learn 机制的认知层。

### 不做的事（为道日损）

- 不重写 evolve.js 学习回路，只在 auto-fix 产出时加 `error_pattern` 字段（补漏不加机制）
- 不新建离线巩固的 cron 任务（需真实数据量积累，遵循 [[feature-validation-rule]]，先建归类体系，离线巩固等数据）
- 不强行接 outcome-learning.md（那是 Phase2 储备，命运已定）
- 归类规则不一次性写满，先建表骨架 + 真实翻车数据的几类，后续按真实 case 扩展
