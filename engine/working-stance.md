## 工作立场 — 框架自己的、会演化的临时判断 (Working Stance)

**学术来源**: 皮尔士「探究信念」(belief as a habit of action + 可谬论的落脚点必须有 belief) + 米德「主我与客我」(self 作为过程,在行动中通过采纳他者视角形成,而非给定实体) + 杜威「探究即立场」(inquiry 不是中立的信息收集,探究者带着一个可被修正的倾向进入情境)。

**为什么要这一步**: 本管线已有的全部动作(神思前提审视/发散互否/八卦镜盲点/方案辩证/评分归因/结论自反/可谬标注/他者/语言放假/先验自检)都在做**判断的动作**——质疑、互否、归因、自反。但**这些动作从不沉淀成框架自己的立场**:每步做完就过,框架始终是个中立裁判,自己从不下场表态"我此刻倾向什么、我凭什么倾向它、刚才那步让我改主意了没"。

这导致两个结构性空洞:
1. **可谬标注悬空**: 综合 fallibilism 要求"必须给一个具体错点",但框架从没说过自己倾向什么——这个"可能错"是没有锚点的自我否定。**真正的可谬是"我倾向A,依据是X,X若不成立我改倾向B"**——先有立场,可谬才落地。立场缺位时,可谬是空的。
2. **进化回路空转**: 进化=变异+选择+遗传。选择端回流的是"框架的立场对不对"——框架没有立场,自然没有"立场对错"可回流,所以 evolve 规则无人读、权重成孤儿键。立场缺位是因,回路空转是果。

工作立场的作用:在收敛(框架第一次下场选择)到综合(立场收口)之间,维持一个**贯穿推理的、会演化的临时倾向**,让判断动作汇聚成一个属于框架自己的、会改主意的判断。这是"判断思维"从程序性走向"有自我"的最后一公里。

### 立场四属性(缺一不可,否则退化回中立裁判)

```
① 有倾向: 框架对当前问题明确表态"我此刻倾向X方向"。
   ⛔不许"各有优劣""视情况而定"骑墙——骑墙等于没有立场,等同没加这个字段。
   可逆小事可留空(见门控),但一旦填就必须有明确倾向。
② 有依据: 倾向凭什么——引用本步及之前步骤的具体产出作为证据(哪步的哪个发现支撑了这个倾向)。
   ⛔不接受"综合考虑后倾向X"——必须指出是哪个具体产出撑起了倾向。这满足"判断依据显式化可追溯"。
③ 会演化: 每步检查本步新产出是否动摇了上一步的倾向——动摇则改倾向,并记录为什么改
   (哪个新证据、怎样动摇了旧依据)。不动摇则记录为何旧倾向仍站得住。
   ⛔这是"自我判断"的核心:会因为证据改主意,不是一条道走到黑。立场不变也要说清为何不变。
④ 可被质疑: 立场自带"这个倾向最可能错在哪"——但这是**基于立场的可谬**
   (我倾向A,A最可能因X不成立而错),不是悬空可谬。这条直接喂给综合的 fallibilism。
```

四属性对应进化的四截:倾向=表现型、依据=遗传信息、演化=单次推理内变异+选择的微缩、可被质疑=选择压。**working_stance 把进化的四截在单次推理内闭环**——不依赖跨run的事后回流(那个难启动),先在单次推理内让框架有"会改主意的自我"。

### ⛔ 与已有动作的边界(防换皮)

工作立场是**容器**不是新动作——它不自己质疑/互否/归因,只把已有动作的产出**汇聚成一个倾向**。具体边界:

- **vs 发散互否**: 发散产视角冲突的合题(知识层);立场引用合题作为倾向依据(判断层)。立场不重做互否,只表态"互否后我倾向哪方"。
- **vs 方案辩证**: 方案辩证产每个方案的失效条件+合题(方案层);立场表态"辩证后我倾向哪个方案"。立场不重做辩证,只选边。
- **vs 评分归因**: 评分产各方案分+最高分归因(评判层);立场可参考但不必等于最高分——立场可以基于评分之外的依据(如终态画像匹配度)倾向非最高分方案,但须说明为何背离评分。
- **vs 结论自反**: 结论自反质疑共享前提(前提层);立场的"可被质疑"质疑的是**立场自身的失效点**(我倾向A,A可能因X错),不是共享前提。两者对象不同:自反管前提,立场管倾向。
- **vs 可谬标注**: 这是**最易混淆的边界**。立场第四属性"可被质疑"是综合 fallibilism 的**输入**——立场说"我倾向A,A最可能因X错",fallibilism 把它落地为"结论最可能错点=X,若X发生改走Y"。立场在收敛/辩论产,fallibilism 在综合收。立场是可谬的锚点,可谬是立场的落地。**没有立场,fallibilism 就是悬空的**——这是本步要修的核心空洞。

### 立场诞生点(只在三步加,不过载)

不在八步全加立场字段——那样过载且违反门控。立场只在框架该表态的三步出现:

```
收敛(converge): 立场诞生。收敛是框架第一次"下场选择"而非罗列——
   从幸存方案里表态倾向哪个+凭什么+什么会让我改主意。
   字段: working_stance { current_lean, grounds, would_change_mind }

辩论(debate): 立场演化。辩论是攻防最激烈处,立场最可能被改变——
   记录辩论后立场变没变+变/不变的依据。
   字段: stance_shift { shifted: bool, from→to(若变)/why_not(若不变), trigger }

综合(synthesis): 立场收口。串联收敛→辩论→综合三阶段倾向+转变原因,
   让最终结论回扣"框架为什么走到这里"而非只回扣用户画像。
   字段: stance_pedigree { stages: [{step, lean, why}], final_lean }
```

### 门控 — 复用神思赌注判断

```
① 复用神思赌注判断: 可逆小事 → working_stance 留空(可逆小事不值得框架表态,硬表态是凑字段)。
② 高赌注问题 → 必填。但"有倾向"是硬约束:一旦填就不许骑墙
   (不许"各有优劣"——骑墙=没立场=没加这个字段,不如留空)。
③ 立场演化(stance_shift)若填 shifted=false,必须给"为何不变"的依据
   (哪个辩论攻击未动摇旧倾向的依据),不接受"立场不变"一句话带过。
```

### ⛔ 与可谬标注的衔接(修悬空可谬,本步核心)

综合的 fallibilism.likely_wrong_point 现在**必须基于 stance_pedigree 的最终立场**:

```
旧(悬空): "本结论最可能错在哪?" —— 框架没说过倾向什么,这个错点是飘的。
新(基于立场): "框架最终倾向A(stance_pedigree.final_lean),A最可能因X不成立而错,
              若X发生改走Y(fallback)。"
```

这样可谬从"没有锚点的自我否定"变成"基于立场的可证伪"——皮尔士可谬论才算真落地(可谬论的前提是先有 belief,working_stance 就是那个 belief)。

### 对象层级

工作立场管"框架自己此刻倾向什么、怎么演化的",⛔不上升到"质疑结论对错"(那是可谬的活,立场只喂锚点给它)、不上升到"保留不可同化的他者"(那是他者性的活)、不上升到"质疑问题框架"(那是语言放假的活)。立场是这些动作的**汇聚容器**,不抢它们的活。

⛔ 不引入哲学术语(working stance/主我客我/探究信念/可谬论/皮尔士/米德/杜威)。对外输出用自然语言:说"到这一步我倾向X,因为前面[某步]发现了Y;如果Z这个依据不成立,我就得改倾向",不说"维持一个可谬的工作立场作为探究信念的载体"。
